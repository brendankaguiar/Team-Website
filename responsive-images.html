Responsive Images

WHAT ARE RESPONSIVE IMAGES?

Responsive images are images that work well on a wide variety of devices with differing screen sizes, resolutions, display density, and other such factors. It is a part of responsive web design that aims to make images universally resposnive and designed no matter what device is used and what size of the viewport it is being displayed in.

Before responsive web design (which includes responsive images) were used, most desktop screens were fixed at 800 or 1024 pixles wide, so websites were made for those sizes.SInce there were only two sizes, widths were already set in the code and didn't change when viewed on a different viewport. Can you imagine all that scrolling?

It wasn't until the early 2000s that developers created "fluid design" and "liquid layout" to fix all the hassles of fixed-width design and make websites more user-friendly on all platforms. Thus, responsive web design (and resposnive images) was born.

Take for example the following scenario. A common website may contain a header image and some content images below the header. It is likely that the header image spans the entire width of the header, with content images inside the body container. Here is an example:

How to Create Basic Responsive Images

Step 1) Add HTML:

<img src="nature.jpg" alt="Nature" class="responsive">

Step 2) Add CSS:

If you want the image to scale both up and down on responsiveness, set the CSS width property to 100% and height to auto.

.responsive {
	width:100%;
	height:auto;
}

If you want ans image to scale down if it has to, but never scale up to be larger than its original size, use "max-width: 100%"

.responsive {
	max-width:100%;
	height:auto;
}

If you want to restrict a responsive image to a maximum size, use the "max-width" property, with a pixel value, em unit, or other sizing unit (except percentages) of your choice:

.responsive {
	width:100%;
	max-width:20vw;
	height:auto;
}

Now if this image were to be set as a header image, the basic responsive principles would take effect, but would not always work for the better. For example, if the responsive header image above is viewed on a smaller screen, the image would look okay, but would start to take up a lot of height, making it difficult to see the rest of the content such as the image. To remedy this issue, we have to utilize something called art direction.

ART DIRECTION

Whenever we need to make changes to content or aspect ratio of an image based on the size of the image in the page, we are utilizing art direction. This displays a cropped version of the image which displays the important details of the image when the site is viewed on a narrow screen. The second cropped image could be displayed for a medium width screen device, such as a tablet. For example, the image below can be cropped to close in on a specific detail.

In addition, there is no need to embed such large images if it being viewed on a mobile screen. We’ve all experienced that issue where we blow up an image that’s way past its original size; it looks grainy, blurry, and ugly. In such a case, one thing we can use to resolve this is called resolution switching. This refers to any scenario where we provide different sizes of an image without making modifications to the content or aspect ratio.

RESOLUTION SWITCHING

Resolution switching is similar to applying art direction; rather than cropping an image to capture the important details of an image, we simply link multiple images that adjusts accordingly to the viewport. In doing so, we can ensure high quality images no matter how small or large a device is. Now, a standard <img> element traditionally only lets us point the browser to a single source file: 

<img src="mountain-top-800w.jpg" alt="Peak of mountain">

So far in this class, we've only sourced images right from their file using the "src" attribute inside an "img" tag. However, we can also use two other attributes -- "srcset" and "sizes" -- to add more source images along with directions to help the browser pick the right one.

The "srcset" attribute defines the set of images we provide for the browser to choose from, along with the size of the image. These various images are typically scaled and cropped using art direction. In addition, the "sizes" attribute defines a set of conditions in the media itself (such as screen width) and indicates which image size would be the best to use in a specific circumstance. Here is an example of integrated resolution switching using both attributes: 

<img srcset="mountain-top-320w.jpg 320w,
	     mountain-top-480w.jpg 480w,
	     mountain-top-800w.jpg 800w"
     sizes="(max-width:320px) 280px,
	    (max-width:480px) 440px,
	    800px"
     src="mountain-top-800w.jpg" alt="Peak of mountain">

With these attrubutes specified, the browser will:

  1. Determine its device width
  2. Go through which media condition in the given sizes is the first to be a true statement
  3. Look at the slot size provided to that specific media query
  4. Load the referenced image in the "srcset" list that best fulfills the specified conditions in the slot size

Therefore, depending on the sizes of the "max-width" property, the browser will automatically adjust which image to display, ensuring the best possible image quality.
